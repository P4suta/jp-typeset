日本語Web組版とGFM

これはMarkdown(GFM)から生成される典型的なHTMLを、jp-typesetで組んだ見本。本文の段落は文頭が1字下げされ、両端揃えになる。和文とHTMLやCSSのような欧文の間には四分アキが入り、重要な語は圏点で強調される。取り消し線やinline code、リンクも自然に混ざる。

見出しレベル2(節)

段落が続く場合、2つ目以降の段落も行頭字下げされる。長い文章を入れて折り返しと禁則を確認する。たとえば「鍵括弧」や、句読点が行頭に来ないこと、約物が適切に扱われることなどが大切である。

見出しレベル3(項)

箇条書き

  • 第一の項目。リスト項目は字下げせず両端揃え。
  • 第二の項目。長めの文を入れて、項目内での折り返しと禁則処理が効くかを確認していく。
  • 入れ子のリスト
    • 子項目その一
    • 子項目その二

番号付きリスト

  1. 最初の手順を実行する。
  2. 次の手順に進む。
  3. 最後に確認する。

タスクリスト

  • 済んだ作業
  • これからの作業

引用

引用は左を2字下げし、行頭字下げはしない。前後にアキを取る。欧文的な左の縦罫線は付けない。

複数段落の引用でも、各段落は字下げされない。

コード

文中のconst 名前 = "値";のようなコードは等幅で、和欧間アキを入れてはいけない。ブロックも同様。 

// 日本語のコメントとcodeが混在する例
function あいさつ(名前) {
  return `こんにちは、${名前}さん`;
}
console.log(あいさつ("世界"));
