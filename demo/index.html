jp-typeset 見本帳

日本語Web組版の標準的な書き方の規則を、対応ブラウザでどこまで実現できるかを並べた一覧。 各カードの枠内がプリセット適用後の描画、下のコードがその HTML ソース。

基本: 段落の字下げ・両端揃え・行間

期待: 段落 p は文頭1字下げ＆両端揃え。見出しは字下げ・両端揃えされない。

ネイティブ: 全エンジン (text-indent / text-align / line-height)

ルビ(ふりがな)

期待: 標準の <ruby> のルビ文字が親文字より小さく整う。

ネイティブ: 全エンジン (標準の ruby/rt)

強調(圏点・傍点)

期待: 和文の <em> は斜体ではなく圏点(ゴマ点)で強調される。

ネイティブ: 全エンジン (text-emphasis)

ブロック引用(和文の作法)

期待: 左を全角2字下げ＋前後アキ。欧文的な左の縦罫線・斜体・鍵括弧は付けない。

ネイティブ: 全エンジン (字下げ方式)

行頭禁則: 句読点

期待: 「、」「。」が行頭に来ない(前行に追い込む)。

ネイティブ: 全エンジン (line-break: strict)

行頭禁則: 小書き仮名・長音符

期待: 「ょ」「っ」「ー」などが行頭に来ない。

ネイティブ: 全エンジン (line-break: strict)

行末禁則: 開き括弧

期待: 開き括弧「が行末に孤立せず次行へ送られる。

ネイティブ: 全エンジン (line-break: strict)

約物詰め: 行頭開き括弧の半角化

期待: 段落・改行直後以外の行頭の開き括弧を半角に詰める。

ネイティブ: Chromium のみ (text-spacing-trim: space-first / halt・chws フォント必須)

約物詰め: 連続する約物

期待: 閉じ括弧と句点の連続「」。などの余分なアキを詰める。

ネイティブ: Chromium のみ (text-spacing-trim)

和欧混植: 和文↔ラテン文字

期待: 和文とラテン語の境界に四分アキが自動で入る。

ネイティブ: 全4エンジン (text-autospace: normal)

和欧混植: 和文↔数字・単位

期待: 和文と半角数字の境界にアキ。数字と単位の間は詰める。

ネイティブ: 全4エンジン (text-autospace: normal)

感嘆符・疑問符

期待: 文末の！？の後ろにアキ。前後の約物が適切に処理される。

ネイティブ: line-break / text-spacing-trim(Chromium)

約物: 三点リーダ・ダッシュ・中黒

期待: 連続するリーダ/ダッシュが分断されず、中黒(・)前後に不自然なアキが入らない。

ネイティブ: 全エンジン (字形は変えず壊さないことの確認)

ぶら下げ: 行末の句読点

期待: 行末に来た「、」「。」が版面右端の外側にぶら下がる。

ネイティブ: Safari のみ (hanging-punctuation: allow-end)

ぶら下げ: 行頭の開き括弧

期待: 行頭の開き括弧「が版面左端の外側にぶら下がる。

ネイティブ: Safari のみ (hanging-punctuation: first)

文節改行: 見出しの不自然な折り返し防止

期待: 「日本語組版」「処理の」などの文節単位で改行される。

ネイティブ: Chromium のみ (word-break: auto-phrase / lang=ja 必須)

文節改行: 狭い幅の段落

期待: 狭幅でも意味のまとまりで折れ、読みやすさを保つ。

ネイティブ: Chromium のみ (word-break: auto-phrase)

縦書き(.jp-vertical)＋縦中横・ルビ・圏点

期待: 縦書きで、禁則・字下げ・両端揃え・ルビ(右)・圏点(右)・縦中横が効く。

ネイティブ: 全エンジン (writing-mode / text-combine-upright)
